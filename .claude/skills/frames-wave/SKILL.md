# FRAMES Wave

Use this skill when planning or updating a FRAMES wave.

## Protocol

1. Read `context/waves/PHASES.md`.
2. Read the active wave `WAVE.md`.
3. Confirm the wave outcome and pulse table.
4. Keep each frame structured as source, target, mapping, action cue, and failure
   mode.
5. Update the wave status and affected pulse notes.
6. Run `git diff --check`.

## Rules

- Do not add manipulative persuasion tactics.
- Do not present analogies as evidence.
- Name where each frame stops working.
- Prefer everyday sources with widely understood behavior.
