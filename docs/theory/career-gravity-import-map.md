## High-Value Imports

These should become first-class FRAMES entries or examples before importing the
whole CAREER set:

1. **Veto Rule**: powerful general frame for averages hiding blockers.
2. **Reset Tax**: useful for decisions involving team changes, tool migrations,
   platform rewrites, and career moves.
3. **Mentor vs. Sponsor**: clean evidence-boundary frame for confusing warmth
   with advocacy.
4. **Portability Index**: useful for any move between contexts where reputation,
   data, trust, or local knowledge may not transfer.
5. **Momentum State**: separates current score from direction of travel.
6. **Advocacy Bandwidth**: useful for managers allocating attention and political
   capital.

## Transfer Maps

### Veto Rule

| Part | Mapping |
|---|---|
| Source scene | A gate with multiple locks. |
| Target situation | A decision with several required dimensions. |
| Transfer | The strongest dimensions do not compensate for the locked one. |
| Action cue | Find the binding lock before optimizing the high scores. |
| Evidence boundary | Verify the dimension is truly required, not just preferred. |
| Failure mode | Can overstate a blocker when the system actually allows tradeoffs. |

### Reset Tax

| Part | Mapping |
|---|---|
| Source scene | Moving to a new field where accumulated gravity resets. |
| Target situation | Changing jobs, teams, tools, vendors, or platforms. |
| Transfer | Some capital transfers; local trust and context must be rebuilt. |
| Action cue | Compare arrival state with current state before moving. |
| Evidence boundary | Verify whether the new context grants a level, sponsor, or preloaded trust. |
| Failure mode | Can make staying look safer than it is when the current system is actively hostile. |

### Mentor vs. Sponsor

| Part | Mapping |
|---|---|
| Source scene | Helpful coach vs. advocate in the room where decisions happen. |
| Target situation | Distinguishing support from power. |
| Transfer | Warmth and advice are not the same as capital spent on your behalf. |
| Action cue | Ask for evidence of unprompted advocacy in absent rooms. |
| Evidence boundary | A sponsor claim requires a concrete instance, not a feeling. |
| Failure mode | Can turn relationships transactional if used without trust. |

### Portability Index

| Part | Mapping |
|---|---|
| Source scene | Luggage: some things travel; some are left behind. |
| Target situation | Moving between organizations, teams, systems, or domains. |
| Transfer | Skills and artifacts may travel; local reputation and context may not. |
| Action cue | List what carries, what resets, and what must be rebuilt. |
| Evidence boundary | Verify what the destination already knows or values. |
| Failure mode | Can undervalue genuinely portable expertise. |

## Import Rules

- Treat CAREER scores as domain-specific examples, not universal evidence.
- Prefer structural frames over numeric formulas in FRAMES docs.
- Add claim-strength labels before public claims about promotion probability.
- Add human-safety warnings for employment-sensitive advice.
- Use plain-language fallback before applying frames to layoffs, managed exits,
  performance actions, or identity-sensitive career situations.
