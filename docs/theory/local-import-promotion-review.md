# Local Import Promotion Review

This review applies the FRAMES role lenses to local imports from RESONANCE
MANAGE, CAREER Gravity, and theme swimlanes. It decides which local patterns can
move from `locally_observed` to a bounded heuristic candidate, and which should
remain held.

This is not catalog acceptance and not empirical validation. It is an
AI-simulated repo role review for stated uses.

## Promoted Heuristic Candidates

### Board Session

Use when: a group needs visible alternatives before commitment.

Role review:

| Role | Finding |
|---|---|
| Frame Fit Reviewer | Pass: action changes from pitch-by-pitch persuasion to side-by-side comparison. |
| Business Leader | Pass: clarifies options, criteria, owner, and decision path. |
| Evidence Boundary Reviewer | Pass with gate: board must include real options and criteria. |
| Misuse Risk Reviewer | Pass with warning: reject if the decision is already fixed. |
| Catalog Structure Reviewer | Promote as draft heuristic candidate, not accepted index entry. |

Promotion note: use as a coordination/decision heuristic when the board changes
the decision process. Do not use it as performative inclusion.

### Contact Structure

Use when: formal org structure does not explain actual information flow,
escalation, trust, or decision visibility.

Role review:

| Role | Finding |
|---|---|
| Frame Fit Reviewer | Pass: distinguishes reporting lines from signal paths. |
| Business Leader | Pass: clarifies escalation, risk, and owner visibility. |
| Evidence Boundary Reviewer | Pass with gate: must rely on observed contact and decision paths. |
| Misuse Risk Reviewer | Revise language: include privacy and purpose boundaries. |
| Catalog Structure Reviewer | Promote as leadership/operations draft heuristic. |

Promotion note: use only with explicit purpose, privacy boundary, and formal
accountability caveat.

### Veto Rule

Use when: a required dimension blocks an otherwise strong option.

Role review:

| Role | Finding |
|---|---|
| Frame Fit Reviewer | Pass: strong relation between required lock and blocked outcome. |
| Business Leader | Pass: prevents averaging away critical blockers. |
| Evidence Boundary Reviewer | Pass with gate: verify required vs. preferred dimension. |
| Misuse Risk Reviewer | Pass with warning: do not use one weak dimension as a biased veto. |
| Catalog Structure Reviewer | Promote as risk/priority draft heuristic. |

Promotion note: strong candidate for future accepted catalog entry after
non-career examples are added.

### Portability Index

Use when: a person, team, tool, product, or system is moving between contexts
and some assets travel while others reset.

Role review:

| Role | Finding |
|---|---|
| Frame Fit Reviewer | Pass: clean transfer from carried assets to rebuilt assets. |
| Business Leader | Pass: clarifies migration, onboarding, and opportunity costs. |
| Evidence Boundary Reviewer | Pass with gate: destination recognition must be checked. |
| Audience Transfer Reviewer | Pass with category requirement: skill, artifact, trust, data, process, relationship, authority. |
| Catalog Structure Reviewer | Promote as transfer/audience draft heuristic. |

Promotion note: good candidate for broader frame packs because it is not
career-specific when translated carefully.

### Momentum State

Use when: current status is misleading without direction, velocity, support, or
resistance.

Role review:

| Role | Finding |
|---|---|
| Frame Fit Reviewer | Pass: adds action value beyond status label. |
| Business Leader | Pass: improves status conversation by naming direction and resistance. |
| Evidence Boundary Reviewer | Pass with gate: trend must be observable. |
| Misuse Risk Reviewer | Pass with warning: do not use optimism about direction to hide current risk. |
| Catalog Structure Reviewer | Promote as momentum/status draft heuristic. |

Promotion note: useful bridge between status frames and momentum frames.

### Theme Swimlanes

Use when: a program needs a small set of contribution lanes tied to one customer
or operating promise.

Role review:

| Role | Finding |
|---|---|
| Frame Fit Reviewer | Pass when lanes change prioritization, sequencing, or exclusion. |
| Business Leader | Pass with gate: each lane needs owner, outcome, measure, and tradeoff. |
| Evidence Boundary Reviewer | Pass with gate: lane label cannot replace outcome evidence. |
| Misuse Risk Reviewer | Pass with warning: reject use as blame, cuts, or coercive alignment. |
| Catalog Structure Reviewer | Promote as leadership/program draft heuristic. |

Promotion note: do not accept every three-theme strategy as a swimlane frame.
The lanes must help say yes and no. The dedicated acceptance gate lives in
[theme-swimlane-role-review.md](theme-swimlane-role-review.md).

## Promotion Rules Confirmed

- Promoted local imports become draft heuristics, not accepted starter entries.
- They stay out of default Rust search until lifecycle, claim-strength, and risk
  metadata exist in the crate.
- Each promoted import needs accepted-catalog review before catalog acceptance.
- Held imports should be used only as draft examples with explicit warnings.

Acceptance review process lives in
[accepted-catalog-review-process.md](accepted-catalog-review-process.md).

  docs catalog, but not accepted starter catalog or default Rust search.
